# UI コンポーネント

## スタイリング

### Tailwind

- style props や css クラスの使用は控え、className で tailwind クラスの使用を優先する
- スタイルのマージにテンプレートリテラルの使用は避け、 cn 関数を使用する
- 任意のカラーの指定は控え、デザインシステムのカラーを優先する
  - 特別な意味や意図のあるカラーはコメントで意図や理由を補足すること
- 複数箇所で使われるデザインの共通化は css のクラスを使うのではなく react コンポーネントとして切り出す
- 中間変数や定数でクラスを持つことは避け、className に直接記載する
  - 例外例：表示要素を配列で持って要素ごとに指定したいクラスを持つ場合
