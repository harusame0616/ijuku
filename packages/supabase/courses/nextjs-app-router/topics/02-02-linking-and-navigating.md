# Link コンポーネントで画面遷移する

## 目標

- ルートレイアウトのヘッダーから `/` と `/posts` に遷移できる
- 記事一覧から各記事詳細に遷移できる
- `<Link>` を使うと自動でプリフェッチされ、クライアントサイド遷移になることを理解する

## 知識

App Router でページ間を移動するには **`next/link` の `<Link>` コンポーネント** を使います。`<Link>` は HTML の `<a>` を拡張したもので、次の 2 つの最適化が自動で働きます。

- **プリフェッチ**: ビューポートに `<Link>` が入った時点で、リンク先のルートを背後で先読みします（**プリフェッチは本番ビルドでのみ有効**で、`next dev` では動きません）
- **クライアントサイド遷移**: 共有レイアウトを保ったまま、ページ部分だけを差し替えるのでフルリロードが起きません

```tsx
import Link from "next/link";

<Link href="/posts">記事一覧</Link>;
```

通常の `<a href="...">` を使うとプリフェッチもクライアントサイド遷移も働かず、毎回フルリロードが発生します。プリフェッチを止めたい場合は `<Link prefetch={false} href="..." />` のように明示的に無効化します（`prefetch={false}` ではビューポート入りでもホバー時でもプリフェッチは行われません）。大量のリンクを並べる無限スクロールなどでは、リソースの節約に有効です。

ナビゲーションが体感的に遅くなる主な原因は、動的ルートに `loading.tsx` が無くてサーバーレスポンスを待つしかないケースと、ネットワークが遅くてプリフェッチが間に合わないケースです。本コースでは後の章で `loading.tsx` を導入してこの問題を解決します。

## ステップ

### 1. ヘッダーにナビゲーションを追加する

`app/layout.tsx` のヘッダーを更新し、`<Link>` でホームと記事一覧に遷移できるようにします。

```tsx
import Link from "next/link";

// ...
<header>
  <strong>
    <Link href="/">Mini Blog</Link>
  </strong>
  <nav>
    <Link href="/posts">記事一覧</Link>
  </nav>
</header>;
```

### 2. 一覧から記事詳細にリンクする

`app/posts/page.tsx` のリストを `<Link>` で各記事詳細にリンクさせます。

```tsx
import Link from "next/link";
import { posts } from "./data";

export default function PostsPage() {
  return (
    <main>
      <h1>記事一覧</h1>
      <ul>
        {posts.map((post) => (
          <li key={post.slug}>
            <Link href={`/posts/${post.slug}`}>{post.title}</Link>
          </li>
        ))}
      </ul>
    </main>
  );
}
```

### 3. クライアントサイド遷移を観察する

開発サーバー（`next dev`）ではプリフェッチは無効ですが、`<Link>` のクライアントサイド遷移自体は開発モードでも動きます。ブラウザの DevTools の Network タブを開いた状態でヘッダーや記事一覧のリンクをクリックしてみてください。ページ全体の再読み込み（フルリロード）が発生せず、必要な部分だけが差し替わって URL が更新されることを確認します。プリフェッチの効果を実際に観察したい場合は、`next build && next start` で本番ビルドを起動し、リンクがビューポートに入った瞬間に背景でリソースが取得される様子を確認してください。

## 完了判定

- ヘッダーから `/` と `/posts` に遷移できる
- 記事一覧から `/posts/hello` などの詳細ページに遷移できる
- 遷移時にブラウザがフルリロードしていないことを確認した

## 補足

`<Link>` は内部的に Client Component なので、`<Link>` を含むコンポーネント自体には `'use client'` を書かなくても使えます。`href` には文字列か URL オブジェクトを渡せます。ハッシュ付きの遷移（例: `/posts#latest`）では Next.js が自動でスクロール位置を調整します。固定ヘッダーがある場合は CSS の `scroll-padding-top` を併用すると、ヘッダーに被らない位置にスクロールできます。

## 理解度チェック

- `<a>` の代わりに `<Link>` を使う主な利点を、ページ遷移時の挙動の観点で説明してください
