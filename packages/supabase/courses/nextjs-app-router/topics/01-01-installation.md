# Next.js プロジェクトを作成する

## 目標

- `create-next-app` を使ってミニブログのベースプロジェクトを作成する
- 開発サーバーを起動し、ブラウザで初期画面を確認する

## 知識

Next.js は React ベースのフルスタックフレームワークで、ファイルベースのルーティングや Server Components を備えています。本コースで扱うのは Next.js 16 系の **App Router** で、`app/` ディレクトリにファイルを置くだけでルーティングが定義される仕組みです。

Next.js 16 を動かすには **Node.js v20.9 以上** が必要です。`node -v` で確認できます。

新しいプロジェクトを始めるときは `create-next-app` という公式 CLI を使います。これは TypeScript・ESLint・Tailwind CSS・App Router・Turbopack といったモダンな構成を一度にセットアップしてくれるコマンドです。本コースでは `--yes` オプションで推奨デフォルトを採用し、TypeScript と Tailwind CSS を有効にした状態で進めます。

`create-next-app` でセットアップした直後の主要なスクリプトは以下の通りです。

- `next dev`: 開発サーバーを起動する。デフォルトのバンドラは Turbopack
- `next build`: 本番用にビルドする
- `next start`: 本番ビルドの結果を起動する

開発サーバーは既定で `http://localhost:3000` を待ち受けます。ファイルを保存するとブラウザに自動反映されます。

## ステップ

### 1. Node.js のインストール状況とバージョンを確認する

このステップは講師（AI）が自動で実施します。学習者の操作は不要です。

1. 講師は `node -v` を実行して Node.js の有無とバージョンを取得する
2. インストール済みかつ `v20.9.0` 以上であれば「OK」と簡潔に伝え、次のステップへ進む
3. 未インストール、またはバージョンが要件未満の場合:
   1. `uname -a`（Windows なら `ver` または学習者へのヒアリング）で学習者の OS を判定する
   2. その時点での **公式の推奨インストール方法を改めて調査する**（Node.js 公式サイト `https://nodejs.org/` のダウンロードページや、各 OS のパッケージマネージャー公式ドキュメントなど）。本トピックには具体的なコマンドを敢えて記載していないので、毎回最新情報を当たること
   3. 学習者の OS と環境（バージョンマネージャーを使いたいか等）を踏まえて、最適なインストール方法を 1 つ推奨として提示する
   4. インストール後に `node -v` を再確認し、要件を満たしていれば次ステップへ進む

### 2. プロジェクトを作成する

このセッションで現在操作している作業ディレクトリで以下を実行し、`mini-blog` という名前で新規プロジェクトを作成してください。講師（AI）が生成されたファイル構成を直接確認しながら進める都合で、講師がアクセスできる場所に作ることが重要です。本コース全体ではここで作ったアプリを「ミニブログ」と呼び、章を進めるごとに機能を追加していきます。

```bash
pnpm create next-app@latest mini-blog --yes
```

`pnpm` 以外を使っているなら `npx create-next-app@latest mini-blog --yes` でも構いません。

### 3. 開発サーバーを起動する

プロジェクトディレクトリに入って開発サーバーを立ち上げます。

```bash
cd mini-blog
pnpm dev
```

ブラウザで `http://localhost:3000` を開き、`create-next-app` が用意した初期ページが表示されることを確認してください。

### 4. ファイル構成をざっと眺める

`app/page.tsx` と `app/layout.tsx` がトップページとルートレイアウトに対応します。試しに `app/page.tsx` の見出しを `Mini Blog` などに書き換えて保存し、ブラウザがホットリロードで更新される様子を確認してみてください。

## 完了判定

- `mini-blog` ディレクトリが生成され、`pnpm dev` でエラーなく開発サーバーが起動する
- `http://localhost:3000` でページが表示される
- `app/page.tsx` を編集すると即座にブラウザに反映される

## 補足

### 本コース全体の前提

本コース全体の前提知識として、以下を最初のトピックである本トピックで一度だけ確認します。

- React の関数コンポーネントと hooks の基本を理解している
- TypeScript の型注釈・基本的な型推論が読める
- HTML / CSS の基本知識
- ターミナル操作（`cd`、コマンド実行）に抵抗がない

これらに不安がある場合は、本コースを進める前に React / TypeScript の入門教材を一度通しておくとスムーズです。本コースは Next.js が主題なので、React や TS そのものの基礎は深掘りしません。

### よくあるつまずき

Node.js のバージョン要件はステップ 1 で講師が自動チェックするのでここでは詳しく触れません。ポート 3000 が他のプロセスで埋まっている場合は、`pnpm dev -- -p 3001` のように別ポートを指定して起動するか、衝突しているプロセスを終了してください。`pnpm` が無いと言われた場合は `corepack enable` で有効化するか、`npx create-next-app@latest mini-blog --yes` のように `npx` 経由でも代替できます。Windows 環境では WSL 上での実行を推奨します。
