# API仕様書: 認証 (Authentication)

## 1. 概要

すべてのAPIエンドポイントはBearerトークンによる認証が必要です。HTTPリクエストの `Authorization` ヘッダーに、Bearer スキームを使用して指定してください。

| ヘッダー名        | 値の形式                | 必須 | 説明                      |
| :---------------- | :---------------------- | :--- | :------------------------ |
| **Authorization** | `Bearer {YOUR_API_KEY}` | Yes  | 発行されたAPIキーを指定。 |

---

## 2. 認証の仕組み

サーバーはユーザーごとに発行したAPIキー（平文）をハッシュ化した値をDBに保持しており、APIキーとユーザーを紐付けて管理しています。

リクエスト受信時の認証フロー:

1. `Authorization` ヘッダーからAPIキー（平文）を取り出す。
2. サーバー側でそのAPIキーを同一アルゴリズムでハッシュ化する。
3. ハッシュ値でDBを検索し、一致するレコードのユーザーを認証済みユーザーとして特定する。
4. 一致するレコードが存在しない場合は `401 Unauthorized` を返す。

---

## 3. エラー

| HTTP Status          | エラーコード (code) | 発生条件 / 理由                     |
| :------------------- | :------------------ | :---------------------------------- |
| **401 Unauthorized** | `UNAUTHORIZED`      | APIキーが未指定、または無効な場合。 |
