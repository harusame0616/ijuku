# API仕様書: 受講開始 (Enroll Course)

## 1. 概要・エンドポイント定義

指定したコースの受講を開始、または再開します。リクエストパラメータに応じて、適切なセクション・トピックのプロンプト情報を返却し、システム上の受講状態を更新します。

| 項目       | 内容                                 |
| :--------- | :----------------------------------- |
| **Method** | `POST`                               |
| **Path**   | `/api/v1/courses/{course_id}/enroll` |
| **認証**   | 要 (APIキー / Bearer Token)          |

---

## 2. リクエスト

### 2.1. パスパラメータ

| パラメータ名 | 型     | 必須 | 説明               |
| :----------- | :----- | :--- | :----------------- |
| `course_id`  | String | Yes  | 受講対象のコースID |

### 2.2. リクエストボディ (JSON)

リクエストボディは省略可能です。省略した場合はレジューム（再開）動作となります。

| パラメータ名 | 型      | 必須 | 説明                                    |
| :----------- | :------ | :--- | :-------------------------------------- |
| `section_no` | Integer | No   | 開始したいセクション番号（1以上の整数） |
| `topic_no`   | Integer | No   | 開始したいトピック番号（1以上の整数）   |

---

## 3. レスポンス

### 3.1. 成功時 (`200 OK`)

指定された条件に基づき、**実際に開始したコンテンツの位置情報**を返却します。`section_no` および `topic_no` はリクエストで指定した値ではなく、実際に開始されたトピックの番号です。

各プロンプトの取得は別途用意されたAPIを使用してください。

```json
{
  "course_id": "AI-101",
  "section_no": 1,
  "topic_no": 1
}
```

### 3.2. エラー時 (4xx)

エラー発生時は、以下のフォーマットでステータスコードとエラー詳細を返却します。

| HTTP Status          | エラーコード (code)      | 発生条件 / 理由                                             |
| :------------------- | :----------------------- | :---------------------------------------------------------- |
| **400 Bad Request**  | `INVALID_PARAMETER`      | `section_no` や `topic_no` が 0以下、または型が不正な場合。 |
| **400 Bad Request**  | `MISSING_SECTION_NUMBER` | `topic_no` は指定されているが、`section_no` が無い場合。    |
| **401 Unauthorized** | `UNAUTHORIZED`           | APIキーが未指定、または無効な場合。                         |
| **403 Forbidden**    | `COURSE_NOT_PURCHASED`   | ユーザーが該当コースの受講権限を持たない場合。              |
| **404 Not Found**    | `COURSE_NOT_FOUND`       | 指定された `course_id` が存在しない場合。                   |
| **404 Not Found**    | `CONTENT_NOT_FOUND`      | 指定されたセクションやトピックが存在しない場合。            |

**エラーレスポンス例:**

```json
{
  "code": "MISSING_SECTION_NUMBER",
  "message": "section_no is required when topic_no is provided."
}
```

## 4. 振る舞い・ロジック

### 4.1. 受講開始位置の決定ロジック

リクエストパラメータの組み合わせにより、以下の優先順位で提供コンテンツを決定します。

| ケース                   | section_no | topic_no | 動作詳細                                            |
| :----------------------- | :--------- | :------- | :-------------------------------------------------- |
| **① レジューム（再開）** | 未指定     | 未指定   | 下記「レジュームロジック」参照。                    |
| **② セクション指定**     | 指定あり   | 未指定   | 指定されたセクションの最初のトピックから開始。      |
| **③ ピンポイント指定**   | 指定あり   | 指定あり | 指定されたセクション・トピックから直接開始。        |

#### レジュームロジック（ケース①の詳細）

そのコース内で「完了」または「開始」ステータスになっているトピックのうち、**セクション番号・トピック番号が最も大きいトピック**を基準として開始位置を決定します。途中のトピックのステータスに関わらず（中抜けがあっても）、番号が最も大きいものを基準とします。

1. **基準トピックが「完了」の場合**: そのトピックの次のトピックから開始します。
   - 次のトピックが同セクション内に存在する場合：同セクションの次のトピック番号から開始。
   - 同セクション内に次のトピックがない場合（最終トピック）：次のセクションの最初のトピックから開始。
   - 最終セクションの最終トピックが完了済みの場合：セクション1・トピック1から開始します（コース先頭に戻る）。
2. **基準トピックが「開始」の場合**: その基準トピックから開始します。
3. **該当するトピックが存在しない場合（未受講）**: セクション1・トピック1から開始します。
